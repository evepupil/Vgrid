# 前端技术选型（弃 AntD → 无头栈）

> 决策日期：2026-07-07 · 状态：已拍板（弃用 AntD）
> 背景：控制台重设计定稿（[`console.html`](console.html)）为「工程仪器/终端」审美，
> AntD 6 自带的 SaaS 后台外观洗不掉，与之冲突；且原型记忆点组件（阶梯图/KPI/模式切换/
> ticker/角标/sparkline）本就全自定义，AntD 只在表单/表格/日期出力。故弃用 AntD，改无头栈：
> 只借「交互行为 + 无障碍」，外观全部自己贴皮。

## 一、选定的库（各司其职）

| 用途 | 选定 | 为什么是它（而非替代） |
|---|---|---|
| 交互原语：下拉/弹窗/气泡/Tab/**模式开关**/滑块/菜单/Toast | **Radix UI Primitives**（`@radix-ui/react-*`） | 最流行、React 优先、零样式、a11y 完善，data-attribute 贴 CSS 最省心。React 19 兼容。替代 React Aria 更完整但 API 更啰嗦、样式模型更重 |
| 数据表格：成交流水 / 扫描 top-N / 策略库 | **TanStack Table**（`@tanstack/react-table`） | 无头，排序/筛选/虚拟滚动自己渲染单元格，天然贴等宽终端表格；与已装的 `react-query` 同门 |
| 日期区间：回测起止 | **react-day-picker**（放进 Radix Popover 里） | Radix **不含** DatePicker，这个是最常用的无头日历，好贴皮 |
| 表单状态 + 校验：策略参数（~15 个 GridConfig 字段）/ 回测参数 | **react-hook-form** | 纯逻辑无外观，校验 + 受控字段一把梭，错误渲染自己写 |
| 图表：净值 / 回撤 / 买入持有 / sparkline | **uPlot** | ~40KB canvas，快、极灵活，全部曲线统一贴琥珀/青主题；替代 lightweight-charts 更贴「交易终端」但会强加它自己的十字线/tooltip 样式，不好统一 |
| 网格阶梯工程图 | **自定义 SVG/DOM** | 原型已经是了，本就没框架的事 |
| 样式 | **原生 CSS + CSS 变量**（沿用原型），按组件拆 CSS Modules | 原型审美已用 CSS 变量表达得很干净，不硬上 Tailwind |

**保留不动**（逻辑非外观）：`@tanstack/react-query`（数据）、`react-router-dom`（路由）、`vite`、`react@19`。

**移除**：`antd`、`@ant-design/charts`。

> 备选整合方案：若想少装几个包，可用 **React Aria Components** 一个库覆盖
> Select/Dialog/Popover/Tooltip/Tabs/**DatePicker**/**NumberField**/**Table**/Switch/Slider，
> 无障碍统一。代价：API 更啰嗦、样式模型（render props + data 属性）上手更陡。
> 当前选 Radix 组合，因手工贴终端皮时摩擦最小。

## 二、坑：Radix Primitives 不包含什么

Radix 只给交互原语，**没有**这些，需上面单配的库补：
- ❌ 日期选择器 → react-day-picker
- ❌ 数据表格 → TanStack Table
- ❌ 表单校验 → react-hook-form
- ❌ 数字输入 → 原生 `<input type=number>` 贴皮，或 react-hook-form 受控 + 校验

## 三、逐组件迁移映射

| 原 AntD | 换成 |
|---|---|
| `Table`（含排序/筛选/分页） | TanStack Table + 自绘单元格 |
| `Form` / `Form.Item` / 校验 | react-hook-form + 自绘错误 |
| `Input` / `InputNumber` | 原生 input 贴皮（受控于 react-hook-form） |
| `Select` | Radix Select |
| `DatePicker` / `RangePicker` | react-day-picker（Radix Popover 内） |
| `Modal` | Radix Dialog |
| `Drawer` | Radix Dialog（side 变体）或自绘 |
| `Tooltip` | Radix Tooltip |
| `Tabs` | Radix Tabs |
| `Dropdown` | Radix DropdownMenu |
| `Switch`（实盘⇄模拟切换） | Radix Switch |
| `Slider` | Radix Slider |
| `message` / `notification` | Radix Toast |
| `@ant-design/charts` 折线/面积 | uPlot |

## 四、落地顺序（与后端 P0 对齐）——建议

后端目前几乎不返回原型要的数据（见 [`../需求/前端驱动需求清单.md`](../需求/前端驱动需求清单.md)，多为 ❌/🟡）。
在真空里重写整套 React 前端 = 对着 mock 数据空转。故走**竖切**，前后端一片一片一起打通：

- **切 0 — 外壳 + 换栈**（无真实数据）：装无头栈、移除 AntD、搭深色终端主题（字体/方格纸底/角标/CSS 变量）、路由、全局实盘⇄模拟盘 Switch、五屏空布局。把原型的「皮」搬进 React 骨架。
- **切 1 — 网格阶梯**（FR-4）：后端结构化阶梯端点 + 前端工程图，端到端。P0 主角。
- **切 2 — 已实现/浮动分离 + 实时报价**（FR-3.5/6.5/11.1）：KPI 块 + ticker 通电。
- **切 3 — 净值/回撤/买入持有**（FR-5.2/5.3）：uPlot 曲线通电。
- 之后按需求清单 P1/P2 逐切。

每切都「后端补端点 + 前端接线」一起做，杜绝前端对 mock、后端无人验。
