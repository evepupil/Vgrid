# Vgrid 路线图（需求基准）

> 本文件是需求与设计的**基准**。实现以此为准，避免跑偏。改需求先改这里。
> 最后更新：2026-07-06

## 一句话

针对银河证券「0.1 元起收 + 万0.5」的超低 ETF 费率，做 T+0 类 ETF（先港股类）的网格量化。
先把**回测 + 模拟盘**跑通，实盘执行层预留可插拔接口。

## 锁定的需求

| 项 | 决定 |
|---|---|
| 标的 | 港股类 T+0 ETF（恒生、恒生科技等），**单标的先跑通**，多标的后续扩展 |
| 资金 | 约 5 万；每格 ≥ 2000 元（费率保底约束）→ 默认格数上限约 25 |
| 费率 | `fee = max(0.1 元, 成交额 × 0.00005)`，ETF 免印花税 |
| 结算 | T+0（当天可买卖来回配对，逻辑最简单） |
| 存储 | SQLite（状态 / 订单 / 成交）+ Parquet（行情缓存） |
| 监控 | 本地 Web 面板 + 消息推送（M4 之后做） |

## 策略设计（核心）

### 网格阶梯（gridlines）
- 基准区间 `[P_low, P_high]`，格数 `N` → `N+1` 条网格线。
- **等差**：`level_i = P_low + i·(P_high−P_low)/N`。
- **等比**：`level_i = P_low·r^i`，`r = (P_high/P_low)^(1/N)`。
- 两种都做，配置项 `spacing_mode` 切换，回测对比。

### 一格一单（lot 语义）
- 在 `level_i` 买入一份，目标在 `level_{i+1}`（上一格）卖出，赚一格差价。
- 每份金额 `per_grid_amount`，份额向下取整到 100 份（1 手）的整数倍。

### 建仓模式（base_build_mode，配置项，回测对比）
- **中枢建仓 CENTER**：启动价 `P0` 上方所有格子的份额，启动时按 `P0` 一次买齐做底仓；下方挂买单。涨上去有货可卖，跌下来有钱可买。
- **零底仓 ZERO**：启动不买，只在下方挂买单，跌下来才逐格建仓。

### 突破处理：非对称追踪网格
- **向上突破**（价格冲出上沿、货卖光）：整个阶梯向上平移追踪。
  - `upper_rebuild_ratio ∈ [0,1]`（参数，回测定）：0 = 只在下方挂买单等回调再买（不追高）；1 = 立即按市价重建底仓；中间按比例。
- **向下突破**（价格跌破下沿）：阶梯向下延伸，但**逐级放大格距 + 调整每格金额**，防止无限补仓 / 黑天鹅。
  - `down_spacing_factor ≥ 1`（参数）：向下每延伸一格，格距 ×该系数（越跌格子越宽，减速下探）。
  - `down_amount_factor`（参数）：向下每格金额的系数（>1 越跌买越多 / =1 不变 / <1 越跌买越少）。
- **资金硬上限** `capital_cap`：占用资金达到上限就停止新买单，兜底黑天鹅。

## 架构分层

纯逻辑（strategy）与 I/O（data/store/执行）严格分离；回测和模拟盘**共用同一份策略引擎**，杜绝「回测一套、实盘另一套」的偏差。执行层做成可插拔（回测撮合器 / 模拟盘虚拟账户 / 未来 QMT 或到价推送）。

```
core/      领域模型（订单、成交、持仓、手续费、网格配置）—— 无 I/O
strategy/  网格阶梯生成 + 引擎状态机 —— 纯逻辑，单测重点
data/      行情下载 + Parquet 缓存                        [M2]
backtest/  分钟线撮合 + 绩效统计                           [M2]
store/     SQLite 持久化                                  [M2+]
report/    绩效报告（对比买入持有、手续费单列）             [M2]
paper/     模拟盘虚拟账户 + 实时轮询                        [M4]
web/       本地监控面板                                    [M4+]
notify/    消息推送                                        [M4+]
cli/       命令行入口                                      [M2]
```

## 里程碑

| 阶段 | 内容 | 产出 |
|---|---|---|
| **M1** | core 领域模型 + strategy 网格阶梯 + 引擎状态机 + 单测 | 策略逻辑可独立验证，门禁全绿 |
| **M2** ✅ | data + backtest + 手续费统计 + 报告 + CLI | 对任意 A 股 / 港股 ETF 跑回测出报告，端到端冒烟通过 |
| **M3** ✅ | 参数扫描（网格搜索） | 穷举旋钮组合按夏普排序找最优；单区间标注样本内 |
| **M4a** ✅ | 模拟盘核心（实时轮询 + 虚拟账户 + SQLite + replay） | 盘中实时跑虚拟账户、断点续跑 |
| M4b | Web 面板（FastAPI + 原生 HTML） | 本地看盘 |
| M4c | 消息推送 | 成交 / 告警通知 |

## 工程约定

- Python 3.12，mypy `strict` + ruff（含 ANN 类型标注）+ pytest，门禁全绿才 commit。
- 金额 / 价格用 `Decimal`，杜绝浮点误差；份额为 `int`（100 的整数倍）。
- 每个大模块在 `docs/模块设计/<模块>.md` 维护归档（设计 / 改动历史 / 实现细节），与实现同步更新。
