# Vgrid 路线图（需求基准）

> 本文件是需求与设计的**基准**。实现以此为准，避免跑偏。改需求先改这里。
> 最后更新：2026-07-08（补充红利 ETF 分红收益对比需求）

## 一句话

针对银河证券「0.1 元起收 + 万0.5」的超低 ETF 费率，做 T+0 类 ETF（先港股类）的网格量化。
先把**回测 + 模拟盘**跑通，实盘执行层预留可插拔接口。网格收益率跑不过定投时，要能用同一套
行情、手续费和指标，把网格、量化定投、买入持有放在一起公平比较。对红利 ETF，还要能把分红、
走势和费用口径放进同一份报告里比较。

## 锁定的需求

| 项 | 决定 |
|---|---|
| 标的 | 港股类 T+0 ETF（恒生、恒生科技等），**单标的先跑通**，多标的后续扩展 |
| 资金 | 约 5 万；每格 ≥ 2000 元（费率保底约束）→ 默认格数上限约 25 |
| 费率 | `fee = max(0.1 元, 成交额 × 0.00005)`，ETF 免印花税 |
| 结算 | T+0（当天可买卖来回配对，逻辑最简单） |
| 存储 | SQLite（状态 / 订单 / 成交）+ Parquet（行情缓存） |
| 监控 | 本地 Web 面板 + 消息推送（M4 之后做） |
| 策略扩展 | 网格之外新增量化定投（DCA）策略，先做回测对比，再考虑模拟盘 / 实盘 |
| 红利 ETF | 批量比较分红、价格走势、分红再投、累计净值和费用口径 |

## 策略设计（核心）

### 策略族
- **网格策略**：适合震荡行情，靠低买高卖赚波动钱，是当前已实现主线。
- **量化定投（DCA）**：适合趋势不确定、想长期摊低成本的场景。它按日期 / 信号分批买入，
  用规则控制每次买多少。
- 两类策略都必须复用同一套行情、手续费、份额取整和绩效指标，避免比较口径漂移。

### 网格阶梯（gridlines）
- 基准区间 `[P_low, P_high]`，格数 `N` → `N+1` 条网格线。
- **等差**：`level_i = P_low + i·(P_high−P_low)/N`。
- **等比**：`level_i = P_low·r^i`，`r = (P_high/P_low)^(1/N)`。
- 两种都做，配置项 `spacing_mode` 切换，回测对比。

### 一格一单（lot 语义）
- 在 `level_i` 买入一份，目标在 `level_{i+1}`（上一格）卖出，赚一格差价。
- 每份金额 `per_grid_amount`，份额向下取整到 100 份（1 手）的整数倍。

### 建仓模式（base_build_mode，配置项，回测对比）
- **中枢建仓 CENTER**：启动价 `P0` 上方所有格子的份额，启动时按 `P0` 一次买齐做底仓；下方挂买单。涨上去有货可卖，跌下来有钱可买。
- **零底仓 ZERO**：启动不买，只在下方挂买单，跌下来才逐格建仓。

### 突破处理：非对称追踪网格
- **向上突破**（价格冲出上沿、货卖光）：整个阶梯向上平移追踪。
  - `upper_rebuild_ratio ∈ [0,1]`（参数，回测定）：0 = 只在下方挂买单等回调再买（不追高）；1 = 立即按市价重建底仓；中间按比例。
- **向下突破**（价格跌破下沿）：阶梯向下延伸，但**逐级放大格距 + 调整每格金额**，防止无限补仓 / 黑天鹅。
  - `down_spacing_factor ≥ 1`（参数）：向下每延伸一格，格距 ×该系数（越跌格子越宽，减速下探）。
  - `down_amount_factor`（参数）：向下每格金额的系数（>1 越跌买越多 / =1 不变 / <1 越跌买越少）。
- **资金硬上限** `capital_cap`：占用资金达到上限就停止新买单，兜底黑天鹅。

## 量化定投设计（新增）

### 第一版范围
- **固定金额定投**：按日 / 周 / 月固定投入 `base_amount`，这是所有增强定投的基准线。
- **跌幅加码定投**：按近期高点回撤分档加码，例如回撤 5% 买 1 倍、10% 买 1.5 倍、20% 买 2 倍。
- **均线偏离定投**：价格低于均线多买，高于均线少买或暂停。大白话：便宜时多买，涨太高时少买。
- **资金上限**：累计投入达到 `cash_cap` 后停止买入，防止回测里无限追加资金。

### 暂不做
- 固定份额定投：现金压力不稳定，第一版价值不高。
- 价值平均策略：需要目标市值曲线和可能卖出，复杂度高，放到第二版。
- 网格 + 定投混合：需要协调底仓、网格仓和现金池，等单独定投回测稳定后再做。
- Web 模拟盘 / 实盘执行：第一版先做离线回测和报告，验证策略价值。

### 信号与成交口径
- 固定日期定投：日期提前确定，用当根 K 线 `open` 成交。
- 跌幅 / 均线信号：只用当前 K 线之前的数据计算信号，下一根 K 线 `open` 成交，避免偷看未来。
- 每笔成交都走 `shares_for_amount` 向下取整到 100 份，并按银河 ETF 费率扣手续费。
- 买不满一手时跳过该次定投，并在回测结果里能看出跳过次数。

### 回测指标
- `invested_amount`：累计投入本金。
- `market_value`：期末持仓市值。
- `cash`：剩余现金。
- `final_equity`：现金 + 持仓市值。
- `profit`：最终资产 - 累计投入。
- `profit_rate_on_invested`：收益 / 累计投入。
- `xirr`：分批投入后的真实年化收益。大白话：每笔钱进场时间不同，用它算综合年化更公平。
- 继续保留最大回撤、手续费、买入次数等指标，方便和网格、买入持有对比。

## 红利 ETF 分红收益对比（新增需求）

详见 [`docs/需求/红利ETF分红收益对比需求.md`](./需求/红利ETF分红收益对比需求.md)。

第一版做离线报告：

- 按名称关键词筛红利 ETF 池（红利 / 高股息 / 红利低波 / 央企红利 / 国企红利）。
- 拉取分红明细、未复权价格、累计净值和费用字段。
- 输出四条收益口径：价格、价格 + 现金分红、价格 + 分红再投、累计净值。
- 管理费 / 托管费 / 销售服务费只展示和解释，真实价格 / 净值口径默认不额外扣，避免重复扣费。
- 横向比较收益、回撤、分红率、费用和数据质量。

## 架构分层

纯逻辑（strategy）与 I/O（data/store/执行）严格分离；回测和模拟盘**共用同一份策略引擎**，杜绝「回测一套、实盘另一套」的偏差。执行层做成可插拔（回测撮合器 / 模拟盘虚拟账户 / 未来 QMT 或到价推送）。

```
core/      领域模型（订单、成交、持仓、手续费、网格配置）—— 无 I/O
strategy/  网格阶梯生成 + 引擎状态机 —— 纯逻辑，单测重点
dca/       量化定投策略：定投日程 + 买入金额规则 + 回测引擎 + XIRR   [M6] ✅
data/      行情下载 + Parquet 缓存                        [M2]
backtest/  分钟线撮合 + 绩效统计 + 策略对比（compare）         [M2/M6] ✅
store/     SQLite 持久化                                  [M2+]
report/    绩效报告（对比买入持有、手续费单列、定投对比）     [M2/M6]
income/    红利 ETF 分红、净值、费用和收益对比                  [M7]
paper/     模拟盘虚拟账户 + 实时轮询                        [M4]
web/       本地控制台（FastAPI API + React 前端）             [M4+/M5]
notify/    消息推送                                        [M4+]
cli/       命令行入口                                      [M2]
```

| **M2** ✅ | data + backtest + 手续费统计 + 报告 + CLI | 对任意 A 股 / 港股 ETF 跑回测出报告，端到端冒烟通过 |

## 工程约定

- Python 3.13，mypy `strict` + ruff（含 ANN 类型标注）+ pytest，门禁全绿才 commit。
- 金额 / 价格用 `Decimal`，杜绝浮点误差；份额为 `int`（100 的整数倍）。
- 每个大模块在 `docs/模块设计/<模块>.md` 维护归档（设计 / 改动历史 / 实现细节），与实现同步更新。
