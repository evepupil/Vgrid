# vgrid-share —— 回测数据 → 知乎分享

把已有回测能力的结果，整理成一篇能直接发知乎的东西（**可能是文章、也可能是回答**，按内容自由发挥，别套死模板）。

> 全局 CLAUDE.md 已经管的（禁「不是…而是…」句式、说人话少术语、严格模式）默认继续遵守，这里不重复。本 skill 只补「做分享」这件事额外要注意的。

## 三步流程

1. **取数**：跑真命令拿到真实数字和图（见「数据纪律」「出图手册」）。
2. **出图**：按要讲的点选图、加 `--chart` 生成 PNG，图落在 `reports/`。
3. **写正文**：拿真数字 + 图，按「写作风格」组织成文章或回答。

---

## ① 数据纪律（最重要）

**代码、数字、结论，全部只能来自真实命令的输出。禁止凭印象编。**

- 报的每个收益率 / 回撤 / 夏普 / 分红金额，都要有对应命令跑出来的出处。拿不准就再跑一遍。
- **ETF 代码必须先确认存在再用**。反面教训：曾经把不存在的 `512890` 当红利 ETF 写进分析，跑出一堆假结论——真实的是 `512880`。列代码前先核（`income compare` 的池子、或东财 spot 表能查到才算数）。
- 缺数就说缺数：某段无日线、无分红、无净值，如实标「数据不全」，别用估算填。
- 引用别人观点或外部数字，标清来源，不跟自己跑出来的回测数混在一起。

**标准取数命令**（都在项目根 `uv run vgrid …`；`--refresh` 强制重下、忽略缓存）：

```bash
# 单标的网格回测报告
uv run vgrid backtest --symbol 510880 --start 2020-01-01 --end 2024-12-31

# 网格 / 定投 / 买入持有 同区间对比
uv run vgrid compare  --symbol 510880 --start 2020-01-01 --end 2024-12-31

# 参数扫描找最优（--spec 指向 fixed+vary 的 JSON）
uv run vgrid scan --symbol 510880 --start 2020-01-01 --end 2024-12-31 --spec spec.json

# 红利 ETF 批量对比（分红 / 走势 / 费用；报告落 reports/）
uv run vgrid income compare --start 2018-01-01 --end 2024-12-31

# 单只红利：策略(定投/网格) + 分红再投增强对照
uv run vgrid income enhance --symbol 510880 --start 2018-01-01 --end 2024-12-31 --strategy dca
```

## ② 出图手册

给上面的命令加 `--chart` 就出可分享 PNG（白底研报风，160 DPI，落 `--out`，默认 `reports/`）。选图看你正文要讲哪个点：

| 图 | 讲什么 | 怎么出 |
|---|---|---|
| **图A 回测主图** | 单策略净值走势 + 买卖点 + 回撤 | `vgrid backtest … --chart` |
| **图C 三方对比** | 网格 vs 定投 vs 买入持有 谁强 | `vgrid compare … --chart` |
| **图D 红利四口径** | 价 / 价+现分 / 价+再投 / 累计净值 差多少 | `vgrid income compare … --chart --chart-top 3` |
| **图E 红利增强** | 分红再投给策略加了多少（绿色填充=分红贡献） | `vgrid income enhance … --strategy dca --chart` |
| **图F 扫描热力图** | 两个参数怎么配最优 | `vgrid scan … --chart`（**扫描规格必须恰好 2 个 vary 维度**，否则跳过不出图） |
| **图B 网格阶梯** | 当前挂单档位分布 | **CLI 未接**，只能代码里调 `vgrid.charts.render_ladder_chart(view, symbol=…)` |

出图要点：
- `income compare --chart-top N` 控出图 ETF 只数（默认前 3）。
- 图函数不改任何回测数，纯画已算好的结果。改配色 / 排版去 `src/vgrid/charts/_style.py`，六张图统一生效。
- 出完在正文里**每张图配一句话说它在讲什么**，别甩图不解释。

## ③ 写作风格（分享场景）

文章还是回答按内容定，不套固定骨架。分享量化回测，守这几条：

- **客观陈述**：只讲数据支持的事实，语气平实。主观套话一律不用——「不是…而是」「先说结论」「拍脑袋」「反直觉」这类词禁用。
- **结果放前面**：开头给出主要结果（哪个策略收益高、差多少），不让读者翻很久才看到。
- **拿数说话**：关键论断后面跟真实数字（收益率 / 回撤 / 分红），能上表就上表，能配图就配图。
- **必带风险提示**：回测都是样本内、历史区间，收尾说明「历史不代表未来、样本内最优实盘未必」。这是底线。
- **说人话**：术语（夏普、回撤、复权、再投）第一次出现配一句大白话解释。读者是知乎普通用户。
- **诚实**：策略跑输、数据不全、假设有局限，照实写。

手动调 `/vgrid-share`。调用后按上面三步走：确认要讲的点 → 跑真命令取数 + 出图 → 组织成文章 / 回答，交给用户发。
