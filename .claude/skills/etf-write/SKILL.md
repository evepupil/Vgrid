# etf-write · ETF 数据回测写文/回答

把 ETF/基金问题做成**数据驱动**的知乎文章或回答：拉真实行情 → 跑回测 → 生图 → 成文。每个结论配数据或图，不空谈、不拍脑袋。

## 两种模式

| 模式 | 触发 | 输入 | 字数上限 | 输出 |
|---|---|---|---|---|
| 写文 | 用户说"写文/写文章" + 给标题 | 标题 | ≤ 2000 字 | md + 配图 |
| 写回答 | 用户说"写回答/回答问题" + 给提问 | 提问 | ≤ 1500 字 | md + 配图 |

用户没明说模式时看输入：给了标题→写文；给了问句→写回答。

## 工作流

1. **拆命题**：从标题/提问提取要验证的核心命题（如"定投 vs 梭哈""持有几年胜率""分红稳不稳"），一句话写下这次要回答什么。
2. **拉数据**：用项目数据层拉真实行情。日线走腾讯 `load_bars`，分红走东财 `fetch_dividends`（必设 NO_PROXY）。详见 [reference.md](reference.md)「数据源」。
3. **跑回测**：复用项目现成引擎（dca / income / charts），临时脚本跑、跑完即删。别每次重写回测逻辑。详见 [reference.md](reference.md)「回测能力」。
4. **生图**：用 `vgrid.charts._style` 样式系统（THEME / title_block / kpi_strip / watermark / save_png），白底专业风 PNG 存 `images/`。别自己另写 matplotlib 样式。详见 [reference.md](reference.md)「生图」。
5. **成文**：写 md，引用 `images/xxx.png`，文末附数据口径 + 免责。严格遵守下面的语言风格和字数。

## 语言风格（强制）

### 黑名单词汇（绝对不用）

- 先说结论 / 先给结论 / 结论先说
- 不是…而是…（含变体"不是 A，是 B""不仅…更是…"）
- 拍脑袋
- 值得一提的是 / 不难发现 / 不难看出 / 显而易见 / 众所周知 / 毫无疑问
- 综上所述 / 总而言之 / 总的来说
- 一图胜千言 / 废话不多说 / 直接上结论 / 干货满满
- 让我们 / 我们一起 / 笔者认为 / 小伙伴们
- 在…的过程中 / 深入探讨

写出来上面任何词，立即改写。

### 风格要求

- **就事论事、平实**：不太口语（不滥用"盘它""梭哈"）、不太书面（不"鉴于""笔者"）。归档感 + 知乎友好。
- **数据先行**：结论配数据或图；没数据的判断标明是判断。
- **诚实标边界**：历史回测写明"过去不代表未来"；价格数据回答不了的（为什么跌、会不会涨）要明说靠基本面/资金面判断，不装能预测。
- **少术语**：必要的术语配一句大白话（如 XIRR=按每笔钱实际进场时间算的真实年化）。

### 字数上限

- 写回答 ≤ 1500 字（正文，不含口径/免责）
- 写文 ≤ 2000 字（看主题可适当伸缩，硬上限 2000）

超了就砍：删重复论点、合并表格、精简举例。图能说清的不再用文字复述。

## 输出规范

- md 文件：项目根，命名 `<主题>-知乎.md`（中文文件名 OK）。
- 配图：`images/` 目录，英文文件名（如 `redchip_01_seesaw.png`），md 里相对路径引用。
- 文末必带两节：「数据口径」（数据源/成交价/手续费/回测方法）+「免责」（历史≠未来、不构成投资建议）。
- 临时回测脚本跑完即删，不留 repo。
